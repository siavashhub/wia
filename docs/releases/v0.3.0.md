# WIA v0.3.0

The third public release of WIA. The headline is **WIA Actions** — a new
top-level surface that turns the weekly briefing into a small, focused set of
suggestions you can draft in one click.

## Highlights

- **WIA Actions (Tier 1 + Tier 2).** A new "Actions" tab next to Briefing and
  Review. WIA inspects this week's entries and suggests two kinds of work:
  - **Follow-up email** — for meetings whose notes mention sending recap /
    minutes / next steps, or whose label looks like a kickoff / debrief /
    retro / planning / QBR. One click opens your mail client with a
    pre-filled subject and body (and copies the body to your clipboard as a
    backup).
  - **Decision note** — for meetings where you wrote "we decided / agreed /
    approved", or whose label looks like a decision / review / approval /
    sign-off. One click saves a Markdown note (`decision-<date>-<slug>.md`)
    with Context / Decision / Alternatives / Next steps / Raw notes
    sections.
- **Status workflow.** Each suggestion can be **accepted**, **snoozed** (one
  week), **dismissed** (won't resurface for the same meeting / week), or
  **completed**. Re-running the scan never overwrites a user-set status — it
  only refreshes cosmetic fields on still-open cards.
- **Drafting is read-only.** Generating a draft doesn't change the action's
  status, so you can preview without commitment.
- **MCP server surface.** `wia-mcp` now exposes `list_actions` and
  `update_action` alongside the existing briefing / entries / export tools,
  so Copilot Chat (or any MCP client) can review and triage actions too.

## What's new

### Actions backend (`wia.core.actions`)

- New `Action`, `ActionKind`, `ActionStatus`, `ActionUpdate` Pydantic models
  in `wia.core.types`.
- New `ActionRow` SQLModel table with a unique `dedupe_key` index, JSON
  payload column, status / priority / snoozed_until / completed_at /
  dismissed_reason. Auto-migrated via the existing `init_db`
  `metadata.create_all`.
- Pure-function suggester registry (`core/actions/registry.py`) with two
  built-ins:
  - `core/actions/follow_up.py` — regex over entry notes (priority 70) and
    label patterns (priority 50).
  - `core/actions/decision_note.py` — regex over entry notes (priority 75)
    and label patterns (priority 55). Label regexes are split between the
    two suggesters so the same entry never produces duplicate cards.
- Suggesters skip manual entries, low/medium-confidence entries, and any
  dedupe key that's already been dismissed.
- The orchestrator runs the registry after `entries_repo.merge_week`. The
  call is best-effort and never blocks the briefing.

### Drafts (`wia.core.actions.drafts`)

- `build_follow_up_email(action) -> EmailDraft(subject, body, mailto)` —
  generates a templated body with Discussion / Decisions / Next steps
  headers and a `mailto:?subject=…&body=…` URL with empty recipients (you
  fill in who).
- `build_decision_note(action) -> NoteDraft(filename, body)` — Markdown
  template plus a date-and-slug filename like
  `decision-2026-05-18-pricing-review.md`.

### HTTP API (`/api/actions`)

| Verb | Path | Purpose |
| --- | --- | --- |
| GET | `/` | List actions for a week. Hides resolved by default; pass `include_resolved=true` for history. |
| GET | `/{id}` | Fetch one. |
| POST | `/{id}/accept` | Move `suggested → accepted`. |
| POST | `/{id}/complete` | Move to `completed`, stamps `completed_at`. |
| POST | `/{id}/snooze` | Body: `{snoozed_until: ISO8601}`. Forces `status=snoozed`. |
| POST | `/{id}/dismiss` | Body: `{reason?: string}`. |
| POST | `/{id}/draft` | Read-only; returns `{kind: "email", subject, body, mailto}` or `{kind: "markdown", filename, body}`. |

### MCP server

- `wia-mcp` now lists two new tools:
  - `list_actions(week_of?, include_resolved?)`
  - `update_action(action_id, status, snoozed_until?, dismissed_reason?)`

### UI

- New "Actions" tab in the header nav with an unread-count badge.
- Per-card kind icon (envelope for follow-up, document for decision note),
  rationale, primary **Draft email** / **Draft note** button, plus
  Accept / Snooze 1w / Dismiss / Mark done.
- "Show resolved" toggle to inspect history without leaving the tab.
- New `document-text` heroicon and a `draftAction()` Alpine method that
  routes email drafts to `mailto:` (with a clipboard backup) and Markdown
  drafts to the pywebview `save_file` bridge (Blob URL fallback when the
  bridge isn't present).

## Compatibility & migrations

- **Schema:** the new `action` table is created automatically on first
  launch. No manual migration step.
- **No auth, network, or settings changes.** WIA still does no M365 sign-in
  itself; auth is delegated to `@microsoft/workiq`.
- **`POST /api/actions/{id}/draft` is intentionally read-only** so it's safe
  to call from any client (including the MCP surface) without side effects.

## What's deferred

- **`stale_reply` suggester.** Requires a raw per-message email store
  (thread IDs, direction, send timestamps). WIA currently only persists
  aggregated `TimeEntry` rows, so this lands in a later release once the
  underlying storage is in place.
- **Tier 3 *executed* actions** (sending mail, filing tasks). Will require
  explicit per-tenant opt-in and is out of scope for v0.3.

## Release pipeline

`version.json`, runs PyInstaller, drives Inno Setup with
`MyAppVersion=0.3.0`, and publishes the GitHub Release.
