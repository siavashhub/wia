# WIA v0.4.2

Maintenance hotfix on top of v0.4.1. No user-visible behaviour
changes — this release refreshes pinned tooling and CI actions picked up by
Dependabot, plus fixes a timezone-dependent test and a release-pipeline
gap that surfaced when tagging.

## Fixes

- **Multi-day all-day calendar split is now timezone-stable.**
  `core.grouping.clamp_long_blocks` treats an end timestamp that lands on
  exact local midnight as the *exclusive* end of the previous day. Without
  this, an Outlook all-day series with `start = Mon 00:00 UTC` and
  `end = Thu 00:00 UTC` produced 3 blocks on Pacific runners but 4 on
  UTC runners, breaking
  `test_clamp_splits_all_day_multi_day_calendar_event` in CI.
- **CI workflow file is well-formed again.** Removed a stray Git merge
  conflict marker from `.github/workflows/ci.yml` (`build-smoke` job)
  that caused every push to `main` to fail with "This run likely failed
  because of a workflow file issue."
